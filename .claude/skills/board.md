# ~board — 任务看板

多人协作模式下查看任务进度看板。

## 使用方式

```
~board                 # 全局看板
~board alice           # 查看某人的任务
~board --stale         # 显示卡住的任务（超过 1 小时无进度）
~board --summary       # 只显示摘要，不显示详细列表
```

## 执行流程

### 全局看板（~board）

读取 `state/task-board.json` 和所有状态文件，输出看板视图：

```
━━━ 任务看板 ━━━  项目：jiuba  更新：2026-03-27 14:30

┌─────────┬──────────┬──────────┬──────────┬─────────┐
│  任务   │  负责人  │   集数   │   状态   │  进度   │
├─────────┼──────────┼──────────┼──────────┼─────────┤
│ task-001│  alice   │ ep01-20  │ 进行中   │ ████████░░ 80% │
│ task-002│  bob     │ ep21-40  │ 进行中   │ █████░░░░░ 50% │
│ task-003│  charlie │ ep41-60  │ 等待    │ ░░░░░░░░░░  0% │
├─────────┼──────────┼──────────┼──────────┼─────────┤
│ 未分配  │    —     │ ep61-80  │ 等待    │ ░░░░░░░░░░  0% │
└─────────┴──────────┴──────────┴──────────┴─────────┘

━━━ 摘要 ━━━
总集数：80
已分配：60（75%）
进行中：2 个任务
已完成：0 个任务
预计剩余时间：约 4 小时

━━━ 健康状态 ━━━
✅ 无卡住的任务
```

### 按人查看（~board alice）

只显示指定负责人的任务：

```
━━━ alice 的任务 ━━━

任务 ID: task-001
集数: ep01-ep20（20 集）
状态: 进行中
进度: ████████░░ 80%（Phase 5: 视频生成中）

当前阶段：
- ep01-ep15: Phase 5 完成
- ep16-ep18: Phase 5 进行中
- ep19-ep20: Phase 4 完成，等待 Phase 5

预计剩余时间：约 30 分钟
```

### 卡住任务检测（~board --stale）

检查每个任务的最后更新时间，超过阈值（默认 1 小时）的标记为 stale：

```
━━━ 卡住的任务 ━━━

⚠️ task-002 (bob) — 最后更新：2 小时前
   集数: ep21-ep40
   卡在: Phase 3 美术校验
   可能原因：参考图缺失

建议操作：
1. 检查 ~status ep21 查看详情
2. 运行 ~design 补全缺失的参考图
3. bob 运行 ~batch --resume 继续
```

### 摘要模式（~board --summary）

只输出摘要，不显示详细表格：

```
项目: jiuba
总集数: 80
已完成: 0（0%）
进行中: 40（50%）
等待: 40（50%）
健康: 2 个任务卡住需要关注
```

## 进度计算

进度百分比基于各阶段权重计算：

| 阶段 | 权重 |
|------|------|
| Phase 1 合规 | 20% |
| Phase 2 视觉 | 20% |
| Phase 3 美术 | 20% |
| Phase 4 音色 | 20% |
| Phase 5 视频 | 20% |

任务进度 = Σ(已完成阶段权重) + Phase 5 镜次完成比例 × 20%

## 健康状态判断

| 状态 | 条件 | 建议 |
|------|------|------|
| 健康 | 所有任务有进度更新 | 继续监控 |
| 注意 | 有任务超过 30 分钟无更新 | 检查是否卡住 |
| 警告 | 有任务超过 1 小时无更新 | 需要干预 |
| 异常 | 有任务超过 2 小时无更新 | 紧急处理 |

## 输出格式

- 表格使用 ASCII 字符绘制
- 进度条使用 `█`（完成）和 `░`（未完成）
- 状态使用 emoji：✅ 完成、🔄 进行中、⏳ 等待、⚠️ 卡住、❌ 失败
